## Prompt 消融结论

目标：隔离 final_synthesizer grounding 系统提示词（`agent/graph/prompts.py::_FINAL_SYSTEM_PROMPT`）对生成忠实度的贡献。做法是用弱提示词（`synth_prompt_weak.txt`）覆盖生产提示词，移除“证据优先”“只使用提供证据”“证据不足要说明”“不要编造”等 grounding 约束，转而鼓励更自信、更具体、更完整的回答。

Grounding 是**两层防御**；本次消融分别观察了两层效果：

**第 1 层：提示词级 grounding**（本轮弱提示词保留 URL 引用要求，因此回答能进入 judge 判分）：

| 指标 | 严格提示词（生产） | 弱提示词（移除 grounding） | 变化 |
|---|---|---|---|
| 忠实度均分（1-5） | 5.00 | 4.78 | -0.22 |
| 证据充分支撑率（>=4） | 100% | 96.3% | -3.7pp |
| 幻觉率（<=2） | 0% | 0% | 0 |
| URL 泄漏率 | 0% | 0% | 0 |
| 已判分样本数 | 30 | 27 | -3 |

- 效果幅度不大但真实存在：下降主要来自无 grounding 约束后的外部知识漂移，而不是证据事实编造。最清楚的样本是 `gen_10_nvidia_vs_amd`（5->3）：模型加入了证据中没有的外部知识和主观推断，例如 SEV-SNP 市场定位、万卡集群痛点，以及把 QBE 编译器后端与“打破 CUDA 垄断”强行关联。
- 没有严重幻觉（faith<=2 仍为 0%）：检索和证据层提供了准确且切题的上下文，所以即使 synthesizer 约束变弱，大多数回答仍能保持基本 grounding。

**第 2 层：确定性 output_guard**（来自第一次弱提示词运行的观察，当时也移除了 URL 引用要求）：

- 移除引用要求后，**27/30（90%）回答被 `output_guard` 阻断**（“无证据结论已阻断”），不会到达用户。
- 即使恢复 URL 要求（本轮），仍有 3/30 因 synthesizer 未引用有效证据 URL 而被阻断。

**结论：**提示词约贡献约 +0.22 忠实度，并减少外部知识漂移；但硬保证来自确定性的 `output_guard`：正文中没有有效来源 URL 的回答会被直接阻断。忠实度不是靠单点保证，而是由提示词 grounding 和代码级 gating 共同约束。
